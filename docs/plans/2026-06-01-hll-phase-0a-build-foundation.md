# Home Layout Legend — Phase 0a: Build Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish a minimal, building, tested, CI-green TypeScript + React + Vite skeleton for Home Layout Legend with Apache-2.0 license. Lays the foundation that subsequent Phase 0 plans build on.

**Architecture:** Single-package web project. TypeScript strict mode. Vite for dev/build. React 18 for UI. Vitest + Testing Library for unit/component tests (with full TDD discipline on application code). Minimal ESLint + Prettier setup (full Clean Code lint rules land in Phase 0d). One GitHub Actions workflow runs typecheck, lint, test, and build on every push and PR.

**Tech Stack:** pnpm 9, Node 20, TypeScript 5.x, Vite 5.x, React 18.x, Vitest 1.x, @testing-library/react 14.x, ESLint 9.x flat config, Prettier 3.x, GitHub Actions.

**Scope boundary:** This plan does NOT include Three.js, R3F, the 6-layer architecture skeleton, the command dispatcher, registries, Three.js renderer, OPFS/IndexedDB scaffolds, ESLint boundary rules, Husky, knowledge graph, Claude agents, the `hll-pack` CLI, Playwright, Storybook, Lighthouse, or any docs beyond a minimal README. Those land in subsequent Phase 0 plans (0b through 0g).

## File Structure

| File                       | Purpose                                                                      |
| -------------------------- | ---------------------------------------------------------------------------- |
| `package.json`             | npm/pnpm manifest; scripts (dev, build, test, typecheck, lint, format); deps |
| `pnpm-lock.yaml`           | dep lockfile (generated by pnpm install — do not edit by hand)               |
| `tsconfig.json`            | TypeScript strict mode config for `src/` and `tests/`                        |
| `tsconfig.node.json`       | TS config for Vite (node context — vite.config.ts)                           |
| `vite.config.ts`           | Vite build config + Vitest integration                                       |
| `eslint.config.js`         | ESLint flat config — TypeScript + React Hooks + React Refresh                |
| `.prettierrc.json`         | Prettier formatting config                                                   |
| `.prettierignore`          | Paths Prettier should skip                                                   |
| `.gitignore`               | (extend existing) ignore `node_modules/`, `dist/`, etc.                      |
| `LICENSE`                  | Apache-2.0 license text                                                      |
| `NOTICE`                   | Apache-2.0 NOTICE template                                                   |
| `README.md`                | Minimal project README                                                       |
| `index.html`               | HTML entry point Vite serves                                                 |
| `src/main.tsx`             | React app entry — mounts `<App />`                                           |
| `src/App.tsx`              | Root App component                                                           |
| `src/App.test.tsx`         | App smoke test (TDD-driven)                                                  |
| `src/setupTests.ts`        | @testing-library setup file (jest-dom matchers)                              |
| `.github/workflows/ci.yml` | CI workflow — typecheck + lint + test + build                                |
| `.nvmrc`                   | Pin Node version for contributors                                            |

Files in this plan are intentionally focused; each has one clear responsibility. Aggregations (e.g., a monorepo workspace structure) are not introduced — the 6-layer source structure lands in Phase 0f.

- [ ] **Step 2: Create `tsconfig.node.json`**

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "lib": ["ES2023"],
    "module": "ESNext",
    "skipLibCheck": true,
    "moduleResolution": "Bundler",
    "allowSyntheticDefaultImports": true,
    "strict": true,
    "composite": true
  },
  "include": ["vite.config.ts", "vitest.config.ts", "eslint.config.js"]
}
```

> **Note:** Do not include `"noEmit": true` here. TypeScript 5+ forbids `noEmit: true` on a composite project referenced from another tsconfig — the constraint surfaces as `TS6310: Referenced project may not disable emit` once real source files exist in `src/`. The composite project must be able to emit declaration files for the references graph; the actual non-emit behavior comes from `pnpm typecheck`'s `tsc --noEmit` flag.

- [ ] **Step 3: Verify tsconfig resolves**

Run: `pnpm exec tsc --showConfig | head -20`
Expected: prints the effective compiler options (target ES2022, strict true, etc.). No errors.
